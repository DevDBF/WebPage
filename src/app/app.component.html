**************************************************************************************************************




Este taller se compone de 4 puntos en los cuales introducimos el análisis de imágenes/videos.

Punto 1: Se realizó las conversiones de escala de grises utilizando el promedio RGB y luma a partir de una imagen, para el desarrollo del promedio RGB se itero por cada de pixel de la imagen para obtener la suma entre R(Red), G(Green) y B(Blue) y su promedio hasta completar el tamaño de pixeles que compone la imagen y para el desarrollo de luma también se itero por cada pixel de la imagen solo que en este caso cada valor R, G y B fue multiplicado con valores menores a uno encontradas en algunas referencias de luma.



Punto 2: Se aplico algunas mascaras de convoluciones para una imagen, para el desarrollo de este punto se creo matrices de tres por tres que indican cada mascara para luego se utilizadas a una función de convolución dando como resultado la máscara según la matriz aplicada.



Punto 3: Para este punto se calculó el histograma con base en la luminosidad, para posteriormente aplicar una implementación del algoritmo Otsu en el cual se calcula una probabilidad , para después calcular el tamaño de cada clase, su media, varianza , todo esto para calcular umbral el cual nos ayudará a generar la imagen segmentada.



Punto 4: Le cambiamos los fps a un video, Para realizarlo usamos una variable de delay la cual nos decía cada cuántos milisegundos se muestra un fotograma. Al correr este punto se verán 4 vieos iguales, el primero es el original a 60fps, el segundo es el video a 15fps, el tercero es el video a 1fps y el cuarto es el video a una velocidad de reproducción x2.

Empezamos aplicando diferentes tipos de iluminación que nos ofrecen las funciones de processing, una vez se entiende como funcionan resulta sencillo aplicarlas. Las aplicamos a un objeto 3D creado con la función sphere()


Para ver la diferencia cuando se aplica iluminación con per-vertex y per-pixel shaders nos apoyamos de este tutorial realizado en OpenGL. Donde se aplican diferentes modelos de iluminación a un objeto 3D creado con la función glutSolidTeapot() y se intercambia entre vertex shader y fragment shader presionando las letras s y f.


Aprovechando la libreria nub de processing implementamos iluminación desde diferentes puntos usando Nodos, apoyados de las funciones pointLight() y setPosition()


Nos interesamos por el modelo de iluminación avanzado "Normal Mapping". Una técnica 3D que permite dar una iluminación y relieve mucho más detallado a la superficie de un objeto usando la matriz de espacio tangente TBN, Pero desafortunadamente no logramos la implementación de este modelo. esta fue la principal fuente con la que nos apoyamos para tratar de realizar la implementación.
